# Workflow: criar tela

1. Confirmar Page Spec, rota, ator, tarefa, dados, ações e permissões. Se a spec faltar, criá-la como parte do escopo.
2. Mapear jornada, arquitetura de informação, conteúdo e estados aplicáveis.
3. Reutilizar tokens/componentes; justificar componente novo.
4. Definir comportamento mobile/tablet/desktop, teclado, foco, semântica, escala de texto e reduce motion.
5. Implementar UI -> controller/provider -> use case -> repository; sem chamadas de infraestrutura no widget.
6. Testar estados, interações, navegação, responsividade e acessibilidade.
7. Executar UI, Architecture e QA Reviews; Security Review em fluxos sensíveis.
8. Atualizar Page Spec, navegação e componentes.
