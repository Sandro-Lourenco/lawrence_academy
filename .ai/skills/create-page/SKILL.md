# Create Page

1. Ler Page Spec, Design System, componentes, acessibilidade, navegação, estado/offline e contratos consumidos.
2. Se a Page Spec não existir, criá-la no mesmo trabalho antes da implementação.
3. Definir objetivo, fluxo, conteúdo, permissões, estados aplicáveis e comportamento responsivo.
4. Implementar com tokens/componentes e separação UI -> Riverpod -> use case -> repository.
5. Validar teclado, foco, semântica, contraste, escala de texto, reduce motion e recuperação de erro.
6. Adicionar testes de widget/provider/navegação e golden somente onde trouxer valor.
7. Atualizar Page Spec e executar UI/Architecture/QA Reviews.
